# AI-Autonomous Project Guidelines (Agent Context)

## 🎯 Strategic Intent
This project is a **Hexagonal + DDD Monorepo**. It is designed for zero-drift between the Rust backend and Next.js frontend, with strict enforcement of business logic isolation.

## 🛠 Architectural Mandates (Strict)
1.  **Domain Purity**: `backend/crates/domain` MUST NOT depend on any other internal crate. Never import `infra`, `api`, or `cli` into `domain`.
2.  **Identity Control**: All domain IDs (e.g., `UserId`) must be **Newtypes** and generated by the backend (Domain/Application layer). Never use raw `Strings`.
3.  **Error Protocol**: Every fallible domain operation MUST return a `DomainError` enum.
4.  **Type Bridge**: Use `ts-rs` with `#[ts(export)]` for all DTOs and Domain structs shared with the frontend.
5.  **Verification**: The source of truth for "Correctness" is `scripts/verify.sh`.

## 🤖 Agentic Workflow & Quality of Life
- **Planning**: Before modifying the `domain` crate, always create a `/plan`.
- **Dependency Management**: Use pinned versions to avoid `edition2024` conflicts (see `Cargo.toml`).
- **Observability**: Use `#[instrument]` from the `tracing` crate for all Use Case and Repository methods.
- **CLI Adapter (Pro-Tip)**: You can execute backend use cases directly via the terminal using `make admin ARGS="..."`. This is the fastest way to test logic without dealing with HTTP!

## 🚀 Port Mapping
- **Frontend**: Port 3000
- **Backend (Web API)**: Port 3001
- **API URL**: Driven by `frontend/.env.local` (`NEXT_PUBLIC_API_URL`).
